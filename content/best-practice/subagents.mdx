# サブエージェント設計

[Subagents - Claude Code Best Practice](https://github.com/shanraisshan/claude-code-best-practice)

## サブエージェントとは

サブエージェントは、独立したコンテキストウィンドウで動作する**自律的アクター**です。メインのClaude Codeセッションとは分離された環境で、カスタムツール、権限、モデル、メモリ、永続的なアイデンティティを持ちます。

**配置場所:** `.claude/agents/<name>.md`

```
.claude/
└── agents/
    ├── security-reviewer.md
    ├── test-writer.md
    └── api-designer.md
```

## 設計原則

### 機能特化型エージェントを作る

汎用的な「QAエンジニア」「バックエンドエンジニア」ではなく、**機能特化型のサブエージェント**を設計します。

```markdown
# 悪い例
- qa-agent.md（汎用QA）
- backend-agent.md（汎用バックエンド）

# 良い例
- auth-security-reviewer.md（認証セキュリティ専門）
- api-schema-validator.md（APIスキーマ検証専門）
- migration-tester.md（マイグレーションテスト専門）
```

機能特化型にすることで、各エージェントに必要なコンテキスト（スキル）を段階的開示で与えられます。

### コンテキスト分離の効果

サブエージェントの最大の利点は**コンテキストウィンドウの分離**です。

- あるエージェントがコードを書き、別のエージェント（同じモデル）がバグを見つける
- メインコンテキストが中間的なツール呼び出しで汚染されない
- 各エージェントが独自の思考空間を持つため、推論品質が向上する

これは「テスト時間計算量（test-time compute）」の活用とも呼ばれます。タスクをオフロードすることで、メインコンテキストの焦点を維持できます。

## 委譲パターン

### 基本パターン：タスクオフロード

複雑な問題に直面した場合、「サブエージェントを使って」と指示することで、より多くの計算リソースを投入できます。

```
> この認証フローのセキュリティレビューをサブエージェントで実行して
```

### コンテキストフォーク

スキルの `context: fork` 設定を使うと、スキルがサブエージェントのコンテキストで実行されます。

```yaml
# SKILL.md 内の設定
context: fork
agent: security-reviewer
```

この場合：
- メインコンテキストは**最終結果のみ**を受け取る
- 中間的なツール呼び出しは見えない
- `agent` フィールドでサブエージェントの種類を指定できる

## 並列作業

### Agent Teams（エージェントチーム）

tmux と git worktree を使って、複数のエージェントが並列で開発を行うパターンです。

```bash
# worktree を作成して並列作業
git worktree add ../feature-auth feature/auth
git worktree add ../feature-api feature/api

# 各 worktree で Claude Code を起動
# tmux session 1: cd ../feature-auth && claude
# tmux session 2: cd ../feature-api && claude
```

各エージェントが独立したブランチで作業するため、コンフリクトを避けながら並列開発が可能です。

### 並列バグ発見

同じコードベースに対して複数のサブエージェントを走らせると、単一エージェントでは見つけにくいバグを発見できます。異なるコンテキストウィンドウが異なる視点を提供するためです。

## エージェント定義ファイルの書き方

```markdown
# Security Reviewer Agent

## 役割
認証・認可フローのセキュリティレビューを行う。

## 権限
- ファイル読み取り: 許可
- ファイル書き込み: レビューレポートのみ
- 外部API: 禁止

## 使用スキル
- security-checklist
- owasp-top10

## 重点領域
- SQLインジェクション
- XSS
- CSRF
- 認証バイパス
```

## ベストプラクティスまとめ

| 項目 | 推奨 |
|------|------|
| エージェント設計 | 汎用ロールではなく機能特化型 |
| コンテキスト | 分離して推論品質を向上 |
| 並列作業 | tmux + git worktree で並列開発 |
| タスク委譲 | 複雑な問題には「サブエージェントを使って」 |
| スキル連携 | `context: fork` で自動的にサブエージェント実行 |
| メインコンテキスト | オフロードで焦点を維持する |
