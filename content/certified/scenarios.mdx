# 試験シナリオ

CCA Foundations試験では6つのシナリオのうち4つが出題されます。各シナリオは実務に基づいた複合問題で、複数のドメインの知識が問われます。

## シナリオ一覧

### シナリオ1: カスタマーサポートエージェント
- **関連ドメイン**: D1（エージェント設計）、D2（ツール設計）、D5（信頼性）
- **内容**: 顧客の問い合わせに自動対応するエージェントの設計
- **ポイント**:
  - エスカレーションフローの設計
  - ツール選択の最適化（FAQ検索、チケット作成等）
  - ハルシネーション防止策

### シナリオ2: コードレビューパイプライン
- **関連ドメイン**: D3（Claude Code）、D1（エージェント設計）、D4（プロンプト）
- **内容**: PRの自動コードレビューシステム
- **ポイント**:
  - Claude Code の CI/CD統合（`-p` フラグ）
  - レビュー基準のプロンプト設計
  - 段階的レビュー（lint → ロジック → セキュリティ）

### シナリオ3: ドキュメント処理ワークフロー
- **関連ドメイン**: D4（構造化出力）、D2（MCP）、D5（コンテキスト管理）
- **内容**: 大量のドキュメントからデータを抽出・構造化
- **ポイント**:
  - 構造化出力（tool_use）の活用
  - 長文ドキュメントのコンテキスト管理
  - Batch APIによるコスト最適化

### シナリオ4: マルチステップリサーチエージェント
- **関連ドメイン**: D1（オーケストレーション）、D5（コンテキスト管理）、D2（ツール設計）
- **内容**: 複数の情報源を調査して報告書を作成するエージェント
- **ポイント**:
  - オーケストレータ-ワーカーパターンの適用
  - Progressive Summarizationでコンテキスト管理
  - 情報来歴の追跡

### シナリオ5: データ分析アシスタント
- **関連ドメイン**: D4（プロンプト）、D2（ツール設計）、D5（信頼性）
- **内容**: データベースに接続して分析・可視化するアシスタント
- **ポイント**:
  - SQLクエリツールの設計と権限管理
  - Few-shotプロンプトでクエリ精度向上
  - 結果のバリデーションとエラーハンドリング

### シナリオ6: コンテンツ生成パイプライン
- **関連ドメイン**: D4（プロンプト）、D1（エージェント設計）、D3（Claude Code）
- **内容**: マーケティングコンテンツの自動生成・管理
- **ポイント**:
  - ブランドガイドラインのシステムプロンプト化
  - レビュー→修正のエージェントループ
  - CLAUDE.mdによるスタイル規約の管理

## 試験対策のコツ

1. **シナリオの横断的理解**: 1つのシナリオで複数ドメインが問われる
2. **トレードオフの判断**: 「最も適切な」選択肢を選ぶ問題が多い
3. **実装の具体性**: 抽象的な理解ではなく、具体的な設計判断が問われる
4. **失敗パターンの理解**: アンチパターンを知っていると消去法が使える
