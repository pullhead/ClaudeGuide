# D4: プロンプト & 構造化出力 (20%)

## プロンプト技法

### Few-shot プロンプティング
- 入出力の例を提示してパターンを学習させる
- 2〜3個の例で十分な場合が多い
- 例の多様性が重要（エッジケースも含める）

### Chain of Thought（CoT）
- 「ステップバイステップで考えてください」と指示
- 推論プロセスを明示的に出力させる
- 複雑な問題で精度が向上
- `<thinking>` タグで思考過程を分離可能（Extended Thinking）

### XMLタグ構造化
- `<instructions>`, `<context>`, `<output>` 等のタグで入力を構造化
- Claudeはネストされたタグ構造を理解する
- 複数の情報源を明確に区別できる

### システムプロンプト
- ロール設定、制約条件、出力フォーマットを定義
- Messages APIの `system` パラメータで指定
- 一貫した動作を保証

### プリフィル（Prefill）
- assistantメッセージの冒頭を指定して出力形式を制御
- 例: `{"result":` でJSON出力を誘導
- 構造化出力の確実性を高める

## 構造化出力

### tool_use による構造化出力
- ツール定義のJSON Schemaに従った出力を強制
- 最も確実な構造化出力の方法
- バリデーションが自動的に行われる

### Validation-Retryパターン
1. Claudeに構造化データを出力させる
2. 出力をバリデーション（スキーマチェック）
3. 失敗した場合、エラーメッセージと共に再試行
4. 最大リトライ回数を設定

### JSONモード
- レスポンスをJSON形式に制限
- `tool_use` より柔軟だが、スキーマ保証が弱い

## コスト最適化

### Batch API
- 大量リクエストを非同期でバッチ処理
- **50%のコスト削減**
- 24時間以内に処理完了
- レイテンシ許容できるユースケースに最適

### Prompt Caching
- 繰り返し使用するプロンプトをキャッシュ
- system プロンプトや長いコンテキストに有効
- **90%のコスト削減**（キャッシュヒット時）
- キャッシュTTL: 5分（最後の使用から）

### トークン効率化
- 不要なコンテキストを削除
- 出力トークン数を制限（`max_tokens`）
- 適切なモデル選択（Haiku < Sonnet < Opus）
