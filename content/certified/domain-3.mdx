# D3: Claude Code (20%)

## CLAUDE.md 階層

CLAUDE.mdはClaude Codeに対するプロジェクト固有の指示ファイル。

### 階層構造と優先順位
1. **`~/.claude/CLAUDE.md`** — グローバル設定（全プロジェクト共通）
2. **プロジェクトルート `CLAUDE.md`** — プロジェクト全体の設定
3. **サブディレクトリ `CLAUDE.md`** — ディレクトリ固有の設定
4. **`.claude/CLAUDE.local.md`** — ローカル個人設定（gitignored）

下位の設定が上位を**オーバーライド**する。

### 書き方のベストプラクティス
- 簡潔で具体的な指示を書く
- コーディング規約、テスト方針、アーキテクチャの方針を記述
- プロジェクト固有のコマンドやワークフローを記述

## Hooks（フック）

Claude Codeの動作をカスタマイズするためのシェルコマンド。

### 5種類のHook
| Hook | タイミング | 用途 |
|------|-----------|------|
| `PreToolUse` | ツール実行前 | バリデーション、ブロック |
| `PostToolUse` | ツール実行後 | 結果の後処理、ログ |
| `Notification` | 通知時 | 外部通知（Slack等） |
| `Stop` | 応答完了時 | 自動検証、追加処理 |
| `SubagentStop` | サブエージェント完了時 | サブエージェントの出力制御 |

### Hook の設定
- `settings.json` で設定
- マッチャーでツール名を指定可能（例: `Edit`, `Write`）
- 終了コード2で操作をブロック

## CI/CD統合

### `-p` フラグ（非対話モード）
- `claude -p "指示"` でCLIから非対話的に実行
- CI/CDパイプラインで自動化に利用
- コードレビュー、テスト生成、ドキュメント更新等

### GitHub Actions連携
- PRのコードレビュー自動化
- Issue対応の自動化
- テスト失敗時の自動修正提案

## 設定の優先順位

高い方から:
1. コマンドライン引数
2. ローカル設定（`.claude/settings.local.json`）
3. プロジェクト設定（`.claude/settings.json`）
4. ユーザー設定（`~/.claude/settings.json`）
5. エンタープライズ設定

## 主要機能
- **タスク管理**: TodoRead/TodoWrite でタスクを追跡
- **サブエージェント**: Agent ツールで並列処理
- **ワークツリー**: 分離された環境で作業
- **MCP統合**: 外部サーバーとの連携

## 試験のポイント
- CLAUDE.md の階層と優先順位
- Hooks の5種類とそれぞれの用途
- `-p` フラグの用途と制限
- 設定ファイルの優先順位
