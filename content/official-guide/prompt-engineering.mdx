# プロンプトエンジニアリング

[Prompt Engineering Overview](https://docs.anthropic.com/en/docs/build-with-claude/prompt-engineering/overview)

## 概要

プロンプトエンジニアリングは、Claudeから最適な出力を得るためにプロンプトを設計・改善する技術。本ページでは主要なテクニックとベストプラクティスをまとめる。

## 前提条件

プロンプトエンジニアリングを始める前に、以下を整備しておく:

1. **成功基準の明確な定義** -- 何をもって良い出力とするか
2. **経験的なテスト方法** -- 基準に対して測定する手段
3. **最初のドラフトプロンプト** -- 改善の出発点

## プロンプトの基本原則

### 明確さと具体性

- 曖昧な指示を避け、具体的に記述する
- 期待する出力フォーマットを明示する
- 制約条件を明確にする

```
悪い例: 「文章を書いてください」
良い例: 「以下のトピックについて、300語程度のブログ記事を書いてください。
対象読者は技術者で、具体的なコード例を含めてください。」
```

### システムプロンプトの活用

```python
message = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    system="あなたはシニアPythonエンジニアです。コードレビューの際は、パフォーマンス、可読性、セキュリティの観点からフィードバックしてください。",
    messages=[
        {"role": "user", "content": "このコードをレビューしてください: ..."}
    ],
)
```

### 長文コンテキストの配置

長いドキュメントやデータは、質問の**前**に配置するとパフォーマンスが向上する。

```
[長いドキュメントの内容]

上記のドキュメントに基づいて、以下の質問に答えてください:
Q: ...
```

## 主要テクニック

### Few-shot プロンプティング

具体例を提供することで、期待する出力パターンを示す。

```
以下の形式でレビューを分類してください。

例1:
入力: 「商品はとても良かったです！」
出力: {"sentiment": "positive", "category": "product_quality"}

例2:
入力: 「配送が遅すぎました」
出力: {"sentiment": "negative", "category": "delivery"}

入力: 「価格は高いが品質は素晴らしい」
出力:
```

### Chain of Thought（段階的思考）

複雑な問題では、段階的に考えるよう指示する。

```
この数学の問題をステップバイステップで解いてください。
各ステップで何をしているか説明してから、次のステップに進んでください。
```

### XMLタグによる構造化

```
<context>
ここにコンテキスト情報
</context>

<instructions>
以下の手順で回答してください:
1. コンテキストを分析する
2. 主要なポイントを3つ抽出する
3. 各ポイントを1文で要約する
</instructions>

<output_format>
- ポイント1: ...
- ポイント2: ...
- ポイント3: ...
</output_format>
```

### ロールプロンプティング

Claudeに特定の役割を与えて、専門的な視点からの回答を引き出す。

```python
system = """あなたはセキュリティ監査の専門家です。
コードを見たら、まずセキュリティの脆弱性を確認し、
OWASP Top 10に基づいてリスクを評価してください。"""
```

### プロンプトチェイニング

複雑なタスクを複数のステップに分解し、各ステップの出力を次の入力にする。

```
ステップ1: ドキュメントから主要なトピックを抽出
ステップ2: 各トピックについて詳細な分析を実施
ステップ3: 分析結果を統合して最終レポートを作成
```

## プロンプト改善のワークフロー

1. **初期プロンプトを作成** -- Claude Consoleのプロンプトジェネレーターも活用可能
2. **テストケースで評価** -- 複数のシナリオで出力を確認
3. **失敗パターンを分析** -- どのケースで期待通りにならないか
4. **テクニックを適用** -- 上記のテクニックから適切なものを選択
5. **反復改善** -- テスト・分析・改善のサイクルを繰り返す

## プロンプトエンジニアリングで解決しない場合

以下の問題はプロンプト以外の手段が有効:

- **レイテンシ** -- より高速なモデルを選択する
- **コスト** -- より安価なモデル、またはPrompt Cachingを使う
- **一貫性** -- [構造化出力](/official-guide/structured-outputs)を使う
- **精度** -- [Extended Thinking](/official-guide/extended-thinking)を有効にする
