# 構造化出力

[Structured Outputs](https://docs.anthropic.com/en/docs/build-with-claude/structured-outputs)

## 概要

構造化出力は、Claudeの応答を特定のスキーマに従わせる機能。2つの方法が提供されている。

1. **JSON Outputs** (`output_config.format`) -- レスポンス全体をJSON形式で制御
2. **Strict Tool Use** (`strict: true`) -- ツール入力のスキーマ検証を保証

## JSON Outputs

### 基本的な使い方

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": "このメールから情報を抽出してください: 田中太郎（tanaka@example.com）がエンタープライズプランに興味を持っています。",
        }
    ],
    output_config={
        "format": {
            "type": "json_schema",
            "schema": {
                "type": "object",
                "properties": {
                    "name": {"type": "string"},
                    "email": {"type": "string"},
                    "plan_interest": {"type": "string"},
                },
                "required": ["name", "email", "plan_interest"],
                "additionalProperties": False,
            },
        }
    },
)
print(response.content[0].text)
# {"name": "田中太郎", "email": "tanaka@example.com", "plan_interest": "エンタープライズ"}
```

### Pydantic / Zod による型安全なパース

**Python（Pydantic）:**
```python
from pydantic import BaseModel

class ContactInfo(BaseModel):
    name: str
    email: str
    plan_interest: str

response = client.messages.parse(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "メールから情報を抽出..."}],
    output_format=ContactInfo,
)
print(response.parsed_output)  # ContactInfo型のオブジェクト
```

**TypeScript（Zod）:**
```typescript
import { z } from "zod";
import { zodOutputFormat } from "@anthropic-ai/sdk/helpers/zod";

const ContactSchema = z.object({
  name: z.string(),
  email: z.string(),
  plan_interest: z.string(),
});

const response = await client.messages.parse({
  model: "claude-sonnet-4-6",
  max_tokens: 1024,
  messages: [{role: "user", content: "メールから情報を抽出..."}],
  output_config: { format: zodOutputFormat(ContactSchema) }
});
console.log(response.parsed_output);
```

## Strict Tool Use

ツール入力のスキーマ準拠を保証する。

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "サンフランシスコの天気は？"}],
    tools=[
        {
            "name": "get_weather",
            "description": "指定場所の天気を取得",
            "strict": True,
            "input_schema": {
                "type": "object",
                "properties": {
                    "location": {"type": "string"},
                    "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]},
                },
                "required": ["location"],
                "additionalProperties": False,
            },
        }
    ],
)
```

### 保証される内容

- ツール `input` は `input_schema` に厳密に準拠
- ツール `name` は常に有効（提供されたツールの中から選択）
- 型の不一致や必須フィールドの欠落がない

## JSON Outputs と Strict Tool Use の併用

エージェントワークフローでは、両方を組み合わせて使える。

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=1024,
    messages=[{"role": "user", "content": "パリへの旅行を計画して"}],
    output_config={
        "format": {
            "type": "json_schema",
            "schema": {
                "type": "object",
                "properties": {
                    "summary": {"type": "string"},
                    "next_steps": {"type": "array", "items": {"type": "string"}},
                },
                "required": ["summary", "next_steps"],
                "additionalProperties": False,
            },
        }
    },
    tools=[
        {
            "name": "search_flights",
            "strict": True,
            "input_schema": {
                "type": "object",
                "properties": {
                    "destination": {"type": "string"},
                    "date": {"type": "string", "format": "date"},
                },
                "required": ["destination", "date"],
                "additionalProperties": False,
            },
        }
    ],
)
```

## 主な利点

- **常に有効なJSON** -- パースエラーが発生しない
- **型安全** -- フィールドの型と必須性が保証される
- **リトライ不要** -- スキーマ違反による再試行が不要
- **制約付きデコーディング** -- 内部的にスキーマに基づいて出力を制約

## 対応モデル

- Claude Opus 4.6 / Sonnet 4.6
- Claude Sonnet 4.5 / Opus 4.5
- Claude Haiku 4.5

## JSON Outputs vs Strict Tool Use の使い分け

| シナリオ | 推奨 |
|:---------|:-----|
| API応答フォーマットの制御 | JSON Outputs |
| データ抽出（構造化レポート等） | JSON Outputs |
| ツール入力の検証 | Strict Tool Use |
| エージェントワークフロー | 両方を併用 |
