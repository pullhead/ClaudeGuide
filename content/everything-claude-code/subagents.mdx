# サブエージェント・並列処理

[Everything Claude Code - Subagents](https://github.com/affaan-m/everything-claude-code)

Claude CodeのAgentツールを使った並列処理、マルチエージェントオーケストレーション、worktreeの活用方法を解説します。

## サブエージェントとは

サブエージェントは、メインのClaude Codeセッションから呼び出される専門的なエージェントです。特定のドメイン（テスト、セキュリティ、レビューなど）に特化し、タスクを分割して効率的に処理します。

## マルチエージェントオーケストレーション

### タスクの分解

```bash
/multi-plan "決済システムを実装"
```

このコマンドは複雑なタスクを複数のサブタスクに分解し、適切なエージェントに割り当てます。

### 並列実行

```bash
/multi-execute
```

分解されたタスクを複数のエージェントで並列に実行します。

### ドメイン別オーケストレーション

```bash
/multi-backend       # バックエンドサービスの並列処理
/multi-frontend      # フロントエンドサービスの並列処理
/multi-workflow       # 汎用マルチサービスワークフロー
/orchestrate         # エージェント間の協調制御
```

## Worktreeの活用

Git worktreeを使うことで、複数のエージェントが同じリポジトリの異なるブランチで同時に作業できます。

### worktreeの基本

```bash
# worktreeを作成
git worktree add ../project-feature-auth feature/auth

# 別のworktreeで別のエージェントを起動
cd ../project-feature-auth
claude
```

### 並列作業のパターン

1. **メインリポジトリ** — 現在の作業を継続
2. **worktree A** — 機能Aを実装するエージェント
3. **worktree B** — 機能Bを実装するエージェント

各worktreeは独立したディレクトリとして存在するため、ファイルの競合なく並列作業が可能です。

## モデル選択によるコスト最適化

サブエージェントのモデルを適切に選択することで、大幅なコスト削減が可能です：

```json
{
  "env": {
    "CLAUDE_CODE_SUBAGENT_MODEL": "haiku"
  }
}
```

| タスク | 推奨モデル | コスト削減率 |
|---|---|---|
| 実装・テスト・デバッグ | Sonnet | 約60% |
| 複雑なアーキテクチャ設計 | Opus | フル性能 |
| サブエージェントのレビュー | Haiku | 約90% |

## 自律ループパターン

ECCの `autonomous-loops` スキルは、以下のパターンを定義しています：

### シーケンシャルパイプライン

タスクを順番に処理するパイプラインです：

```
計画 → 実装 → テスト → レビュー → デプロイ
```

### PRベースループ

PRの作成・レビュー・修正を繰り返すループです：

```
コード変更 → PR作成 → レビュー → 修正 → マージ
```

### DAGオーケストレーション

依存関係のあるタスクをDAG（有向非巡回グラフ）として定義し、可能な限り並列に実行します。

## 反復的コンテキスト取得（Iterative Retrieval）

サブエージェントへのコンテキスト受け渡しを最適化するパターンです：

```bash
# iterative-retrieval スキルを使用
```

段階的にコンテキストを精緻化することで、サブエージェントに渡すトークン量を最小化します。これにより：

- サブエージェントのコストを削減
- コンテキストウィンドウの有効活用
- より正確な応答の生成

## PM2統合

PM2を使って複雑なマルチサービスワークフローを管理できます：

```bash
/pm2    # PM2サービスコマンドを生成
```

プロセスのライフサイクル管理を自動化し、複数のサービスを協調して動作させます。

## ベストプラクティス

1. **適切な粒度でタスクを分割する** — 細かすぎると通信オーバーヘッドが増え、粗すぎると並列化のメリットが減る
2. **サブエージェントには軽量モデルを使う** — レビューや簡単なタスクにはHaikuを使ってコストを抑える
3. **worktreeでファイル競合を防ぐ** — 同じファイルを複数のエージェントが同時に編集しない
4. **チェックポイントを活用する** — `/checkpoint` で状態を保存し、エラー時に復元できるようにする
5. **オーケストレーションを使い分ける** — 独立タスクは `/multi-execute`、協調が必要なタスクは `/orchestrate`
