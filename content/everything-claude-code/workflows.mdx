# ワークフロー・オーケストレーション

[Everything Claude Code - Workflows](https://github.com/affaan-m/everything-claude-code)

Claude Codeを使った実践的な開発ワークフローとオーケストレーション手法を解説します。

## 新機能の開発ワークフロー

最も一般的なワークフローです。計画、TDD、レビュー、セキュリティチェックの順で進めます。

```bash
# 1. 実装計画を作成
/plan "OAuth認証を追加"

# 2. テスト駆動開発（テストを先に書く）
/tdd

# 3. コードレビュー
/code-review

# 4. セキュリティ監査
/security-scan
```

### 流れの詳細

1. **`/plan`** — plannerエージェントが機能の実装ブループリントを作成。依存関係、影響範囲、実装順序を整理
2. **`/tdd`** — tdd-guideエージェントがテストファースト開発を強制。失敗するテストを先に書き、実装で通す
3. **`/code-review`** — code-reviewerエージェントが品質とセキュリティを確認
4. **`/security-scan`** — AgentShieldがOWASP Top 10に基づく脆弱性スキャンを実行

## バグ修正ワークフロー

```bash
# 1. バグを再現するテストを作成
/tdd

# 2. 修正を実装してテストが通ることを確認

# 3. リグレッションチェック
/code-review
```

テスト駆動のバグ修正により、同じバグの再発を防ぎます。

## プロダクションリリース準備

```bash
# 1. セキュリティ監査（OWASP Top 10）
/security-scan

# 2. クリティカルなユーザーフローのE2Eテスト
/e2e

# 3. テストカバレッジ確認（80%以上）
/test-coverage

# 4. 最終品質チェック
/quality-gate
```

## マルチエージェントワークフロー

大規模なタスクを複数のエージェントで分担して並列処理します。

### 計画フェーズ

```bash
/multi-plan "決済システムを実装"
```

タスクを以下のように分解します：
- API設計 → architectエージェント
- データベーススキーマ → database-reviewerエージェント
- 認証・認可 → security-reviewerエージェント
- フロントエンド → code-reviewerエージェント

### 実行フェーズ

```bash
/multi-execute          # 分解されたタスクを並列実行
/multi-backend          # バックエンド専用
/multi-frontend         # フロントエンド専用
```

### 協調制御

```bash
/orchestrate            # エージェント間の協調が必要な場合
```

純粋な並列化ではなく、エージェント間でコンテキストを共有しながら進める必要がある場合に使用します。

## 検証ループ

### 包括的な検証

```bash
/verify
```

以下を一括で実行します：
- ビルドコンパイル
- テスト実行
- リント
- 型チェック
- セキュリティスキャン

### 品質ゲート

```bash
/quality-gate                    # 全体チェック
/quality-gate --path src/auth    # 特定パスのみチェック
```

問題のあるコードがマージされるのを防ぎます。

### カスタム評価

```bash
/eval "パフォーマンス仕様を満たしているか"
```

カスタム基準に対する評価が可能です。

## 継続学習ワークフロー

セッションから自動的にパターンを学習し、改善を蓄積するワークフローです。

```bash
# セッション中にパターンを抽出
/learn

# 評価してから保存
/learn-eval

# 学習済みインスティンクトの確認
/instinct-status

# インスティンクトをスキルに進化
/evolve
```

### チームでの共有

```bash
# インスティンクトをエクスポート
/instinct-export

# チームメンバーからインポート
/instinct-import patterns-2025.md
```

チーム全体の知見を共有・蓄積できます。

## セッション管理

```bash
# セッション履歴を表示
/sessions

# 論理的な区切りで状態を保存
/checkpoint

# 検証基準に対するチェック
/verify
```

長時間のセッションでは、定期的に `/checkpoint` で状態を保存することで、問題発生時に復元できます。

## ワークフローの組み合わせ例

### 日常開発の基本フロー

```bash
/model sonnet              # コスト効率の良いモデルで開始
# ... 実装作業 ...
/compact                   # マイルストーンでコンパクション
/verify                    # 検証
/checkpoint                # 状態保存
# ... 次のタスク ...
/cost                      # コスト確認
```

### 大規模リファクタリング

```bash
/plan "認証システムをリファクタリング"
/multi-plan "認証リファクタリング"    # マルチエージェントで分解
/multi-execute                        # 並列実行
/verify                               # 一括検証
/security-scan                        # セキュリティ確認
/test-coverage                        # カバレッジ確認
```

### 緊急バグ修正

```bash
/model opus                # 複雑なバグにはOpus
/tdd                       # 再現テスト作成
# ... バグ修正 ...
/verify                    # 検証
/model sonnet              # コスト効率モデルに戻す
```
