# 学習トラック: Claude for Work

[Claude for Work - Anthropic Learn](https://www.anthropic.com/learn/claude-for-work)

## 概要

「Claude for Work」は、**組織全体で Claude を導入し、チームの生産性を最大化する**ための学習トラックです。Claude の基本から職種別のユースケース、各種機能の活用方法、推奨コースまでを9つのトピックで体系的にまとめています。

**主要リンク:**
- [Claude for Teams](https://www.anthropic.com/teams) — チーム向けプラン
- [デモをリクエスト](https://www.anthropic.com/contact-sales) — 組織導入の相談

### 4. Artifacts（アーティファクト）

Claude との対話中に**ファイルやコンテンツを作成・管理・共同編集**する機能です。コーディングの知識がなくても、視覚的なコンテンツや構造化されたドキュメントを作成できます。

#### 主な機能
- **ドキュメント生成** — レポート、企画書、マニュアルなどの文書作成
- **コード生成** — HTML/CSS/JavaScript によるインタラクティブなウェブアプリのプロトタイプ
- **表・図の生成** — データの表形式表示、フローチャート、ダイアグラム
- **リアルタイム編集** — 生成物を会話の中で繰り返し修正・改善
- **ダウンロード・共有** — 完成したアーティファクトのエクスポート

#### リソース
- Artifacts の使い方ガイド

#### 活用のポイント
Artifacts は「Claude に作ってもらう」際に特に威力を発揮します。文章だけでなく、レイアウトを持ったドキュメントやインタラクティブなツールを素早くプロトタイピングできるため、チーム内のコミュニケーションやアイデア共有に最適です。

---

### 5. Projects（プロジェクト）

作業を整理し、複雑なタスクで**共同作業する**ための機能です。プロジェクト内にコンテキスト情報を集約することで、Claude が一貫した理解を持って応答します。

#### 主な機能
- **プロジェクトの作成と管理** — テーマ別にワークスペースを整理
- **ファイルアップロード** — 社内ドキュメント、データ、仕様書などをアップロードしてコンテキストを設定
- **カスタム指示** — プロジェクト固有のルールやスタイルガイドを設定
- **チーム共有** — プロジェクトをチームメンバーと共有して共同利用
- **複数会話の統合** — 関連する複数の会話を1つのプロジェクトにまとめる

#### リソース
- Projects の使い方ガイド

#### 活用のポイント
社内のスタイルガイド、ブランドガイドライン、技術仕様書などをプロジェクトにアップロードしておくことで、Claude は常にそれらを参照しながら回答します。チーム全体で一貫した品質のアウトプットを得るための基盤となります。

### 6. Skills（スキル）

詳細な指示を提供することで、**特定のタスクをより正確かつ一貫して実行**させる機能です。定型的な業務を高品質に自動化するための仕組みです。

#### 主な機能
- **Skills の概要** — Skills の仕組みと活用場面
- **Claude での Skills の使い方** — 既存 Skills の選択と適用
- **カスタム Skills の作成** — 組織固有のタスクに最適化した Skills の設計
- **チーム内共有** — 作成した Skills テンプレートをチームメンバーと共有

#### 活用例
- 議事録のフォーマット統一
- 社内メールの文体統一
- コードレビューのチェックリスト適用
- レポートの定型テンプレート出力

## 組織導入のステップ

### フェーズ 1: 基盤構築
1. **個人での習熟** — Claude 101 コースでチームメンバー全員が基本を理解
2. **モデル理解** — Meet the Models でタスクに応じたモデル選択を学ぶ

### フェーズ 2: ユースケース展開
3. **ユースケース特定** — Ways to Use Claude を参考に各部門での具体的な活用シーンを洗い出す
4. **Artifacts & Projects** — チーム共通のプロジェクトを設定し、コンテキストを整備

### フェーズ 3: 標準化と最適化
5. **Skills の整備** — 部門共通の Skills テンプレートを作成・共有
6. **ツール連携** — GitHub、Google Docs 等の既存ツールとの統合を設定
7. **リサーチ活用** — Research 機能による情報収集の効率化

### フェーズ 4: 継続的改善
8. **ベストプラクティスの共有** — 成功事例をチーム内で展開
9. **新機能のキャッチアップ** — 新しいモデルや機能の導入を段階的に進める
