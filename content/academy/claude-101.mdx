# Claude 101

[Claude 101 - Anthropic Academy](https://anthropic.skilljar.com/claude-101)

## コース概要

Claude 101 は、Claude を日常の業務タスクに活用するための**エンドユーザー向け入門コース**です。Claude の基本的な使い方、コア機能の理解、さらに高度な学習リソースへのナビゲーションまでをカバーします。Claude を初めて使う方や、すでに使っているが機能を十分に活用できていない方に最適です。

コースは以下の9つのトピックで構成されており、所要時間は合計で約2〜3時間です。各トピックは独立しているため、興味のあるトピックから始めることも可能です。

---

## トピック 1: Your first conversation（最初の会話）

### 何を学ぶか

Claude との最初の会話を始めるためのすべての基礎を学びます。アカウントの作成からインターフェースの操作、さまざまな入力方法まで、Claude を使い始めるために必要な知識を網羅します。

### アカウント作成と開始手順

1. **claude.ai にアクセス**する — ブラウザで [claude.ai](https://claude.ai) を開きます
2. **アカウントを作成**する — メールアドレスまたは Google / Apple アカウントでサインアップします
3. **プランを選択**する — 無料プラン（Free）でもすぐに使い始められます。より多くの機能や高い利用上限が必要な場合は、Pro / Max / Team / Enterprise プランが用意されています

### チャットインターフェースの基本

Claude のインターフェースは、シンプルなチャット形式です。

- **メッセージ入力欄**: 画面下部のテキストボックスにメッセージを入力し、Enter キーまたは送信ボタンで送信します
- **会話履歴**: 左側のサイドバーに過去の会話が一覧表示されます。タイトルをクリックして過去の会話に戻れます
- **新しい会話**: 左上の「New chat」ボタンで新しい会話を開始します
- **モデル選択**: 会話開始時に使用するモデル（Claude Opus、Claude Sonnet、Claude Haiku など）を選択できます。モデルによって性能・速度・コストが異なります

### さまざまな入力方法

Claude はテキストだけでなく、さまざまな形式の入力を受け付けます。

- **テキスト入力**: 通常の質問やリクエストをテキストで入力します
- **ファイルアップロード**: PDF、Word、Excel、テキストファイル、画像などをアップロードして、その内容について質問したり分析を依頼したりできます。クリップアイコンまたはドラッグ＆ドロップでアップロードします
- **画像の入力**: スクリーンショットや写真を貼り付けて、画像の内容を分析・説明させることができます（Vision 機能）
- **コードの貼り付け**: コードスニペットを貼り付けて、レビュー、デバッグ、説明を求めることができます

### 会話の管理

- **会話のリネーム**: サイドバーの会話タイトルを右クリックまたは三点メニューから名前を変更できます
- **会話の削除**: 不要な会話を削除して整理できます
- **会話の検索**: サイドバー上部の検索機能で過去の会話を素早く見つけられます

### 実践のポイント

- まずは気軽に話しかけてみましょう。「こんにちは」からでも構いません
- Claude は文脈を覚えているので、同じ会話内では前の発言を踏まえた回答を返します
- 長いドキュメントを分析させたい場合は、ファイルをアップロードするのが最も効率的です
- 回答が期待と異なった場合は、追加の指示やフィードバックを送って改善できます

## トピック 2: Getting better results through prompting（プロンプトのコツ）

### 何を学ぶか

Claude からより良い回答を引き出すためのプロンプトエンジニアリング（プロンプト技法）の基礎を学びます。同じ質問でも、聞き方を変えるだけで回答の質が大きく変わります。ここでは実践的なテクニックを体系的に身につけます。

### 基本原則 1: 明確で具体的に伝える

Claude は明確で具体的な指示に最もよく応答します。曖昧な依頼ではなく、何を求めているかを正確に伝えましょう。

**黄金ルール**: あなたのプロンプトを、背景知識のない同僚に見せてください。もし同僚が混乱するなら、Claude も同様に混乱します。

| 改善前 | 改善後 |
|--------|--------|
| 「分析ダッシュボードを作って」 | 「売上データの分析ダッシュボードを作ってください。月次推移グラフ、商品カテゴリ別の売上比較、前年同期比を含めてください」 |
| 「この文章を直して」 | 「この文章をビジネスメールにふさわしいフォーマルなトーンに書き直してください。要点を3つの段落にまとめてください」 |

### 基本原則 2: コンテキスト（背景情報）を提供する

指示だけでなく、「なぜそうしてほしいのか」という背景情報を提供すると、Claude はより的確な回答を生成します。

| 改善前 | 改善後 |
|--------|--------|
| 「省略記号は絶対に使わないで」 | 「あなたの回答はテキスト読み上げエンジンで読み上げられます。省略記号は読み上げエンジンが発音できないので使わないでください」 |

背景を伝えることで、Claude は指示の意図を理解し、類似の状況にも適切に対応できるようになります。

### 基本原則 3: 例を示す（Few-shot プロンプティング）

期待する出力のフォーマットやスタイルを伝える最も効果的な方法は、具体例を示すことです。

- **3〜5個の例**を示すのが最も効果的
- 例は実際のユースケースに近いものにする
- エッジケースも含めて多様な例を提供する
- 例と指示を区別するために、明確なラベルを付ける

```
以下の例に従って、製品レビューを要約してください。

例1:
レビュー: 「バッテリーの持ちが良く、画面もきれい。ただし重い。」
要約: ポジティブ — バッテリー持続力と画面品質を評価。重量が唯一の不満点。

例2:
レビュー: 「配送が遅れたが、商品自体は問題なし。」
要約: 中立 — 商品品質は問題ないが、配送遅延に不満。

では、以下のレビューを要約してください:
レビュー: 「...」
```

### 基本原則 4: 出力フォーマットを指定する

Claude に期待する回答の形式を明示的に伝えましょう。

- **箇条書き/番号付きリスト**: 「3つのポイントを箇条書きで挙げてください」
- **表形式**: 「比較表を作成してください」
- **段階的な説明**: 「ステップバイステップで説明してください」
- **文章のトーン**: 「カジュアルなトーンで」「専門家向けに」「小学生にもわかるように」

### 基本原則 5: 役割を設定する

Claude に特定の役割を与えることで、回答のトーンや専門性を調整できます。

```
あなたは10年の経験を持つシニアデータアナリストです。
以下のデータセットを分析し、経営陣向けのレポートを作成してください。
```

### 基本原則 6: 構造化されたプロンプトを使う

複雑な指示の場合、XMLタグなどで構造化すると効果的です。

```xml
<指示>
以下のドキュメントを分析し、要約を作成してください。
</指示>

<フォーマット>
- 3文以内の要約
- 主要なポイント3つ
- 推奨アクション
</フォーマット>

<ドキュメント>
（ここにドキュメントの内容）
</ドキュメント>
```

### 基本原則 7: 段階的に改善する（反復プロンプティング）

一度のプロンプトで完璧な結果を得ようとせず、会話を通じて段階的に改善していくアプローチも有効です。

1. まず初回の指示を送る
2. 結果を確認し、改善点をフィードバックする
3. 「もっと詳しく」「トーンをもう少しカジュアルに」「この部分を書き直して」と具体的に修正を依頼する
4. 満足のいく結果が得られるまで繰り返す

### 実践のポイント

- **長い文書を扱う場合**: ドキュメントをプロンプトの上部に配置し、質問や指示を下部に書くと精度が向上します
- **複数のタスクがある場合**: 一度に全部を依頼するよりも、1つずつ順番に依頼する方が質の高い結果が得られることがあります
- **「やめてほしいこと」ではなく「やってほしいこと」を伝える**: 「マークダウンを使わないで」より「滑らかな散文で回答してください」の方が効果的です

## トピック 3: The Claude desktop app（デスクトップアプリ）

### 何を学ぶか

Claude デスクトップアプリのインストール方法、基本的な使い方、ブラウザ版との違い、デスクトップならではの活用方法を学びます。

### インストール手順

1. [claude.ai/download](https://claude.ai/download) にアクセスします
2. お使いの OS（macOS / Windows）に対応したインストーラをダウンロードします
3. ダウンロードしたファイルを実行してインストールを完了します
4. アプリを起動し、claude.ai のアカウントでログインします

### デスクトップアプリの主な特徴

**キーボードショートカットでの起動**

デスクトップアプリを使うと、どのアプリケーションで作業していても、グローバルキーボードショートカットで素早く Claude を呼び出せます。

- **macOS**: `Option + Space` で Claude をすぐに開ける
- **Windows**: `Alt + Space` で Claude をすぐに開ける

ブラウザのタブを探す手間がなく、作業の流れを中断せずに Claude に質問できます。

**常駐型のアクセス**

タスクバーまたはメニューバーに常駐するため、いつでもワンクリック（またはショートカットキー）で Claude にアクセスできます。ブラウザを開いていなくても使えるのが大きな利点です。

**ブラウザ版と同じ機能**

デスクトップアプリは Web 版と同じ機能を備えています。

- チャット機能
- ファイルアップロード
- プロジェクト管理
- Artifacts の表示と操作
- 会話履歴の管理
- モデルの選択

### ブラウザ版との違い

| 項目 | ブラウザ版 | デスクトップアプリ |
|------|-----------|-------------------|
| アクセス方法 | ブラウザでclaude.aiを開く | アプリを起動またはショートカットキー |
| 常時起動 | タブを開いておく必要あり | メニューバー/タスクバーに常駐 |
| ショートカット | なし | Option+Space (Mac) / Alt+Space (Win) |
| オフライン | 不可 | 不可（インターネット接続が必要） |
| インストール | 不要 | 必要 |

### 実践のポイント

- テキストエディタやスプレッドシートで作業中に、ショートカットキーで Claude を呼び出して質問し、すぐ元の作業に戻るという使い方が便利です
- ブラウザのタブが多くなりがちな方には特におすすめです
- デスクトップアプリとブラウザ版で会話履歴は同期されるので、どちらからでも同じ会話にアクセスできます

## トピック 4: Projects（プロジェクト機能）

### 何を学ぶか

プロジェクト機能を使って、関連する会話やファイルを整理し、チームで共有する方法を学びます。繰り返し使うワークフローを効率化し、Claude に一貫したコンテキストを与えるための強力な機能です。

### プロジェクトとは

プロジェクトは、Claude での作業を「テーマ」や「タスク」ごとにまとめる整理機能です。プロジェクトの中に知識ベース（ファイルやドキュメント）とカスタム指示を設定することで、そのプロジェクト内のすべての会話で一貫したコンテキストが自動的に適用されます。

例えば「製品マーケティング」というプロジェクトを作成し、ブランドガイドライン、製品仕様書、ターゲット顧客の情報をアップロードしておけば、そのプロジェクト内でのすべての会話で Claude がこれらの情報を参照して回答を生成します。

### プロジェクトの作成手順

1. **サイドバーの「Projects」セクション**を開きます
2. **「Create project」をクリック**して新しいプロジェクトを作成します
3. **プロジェクト名と説明**を入力します（例: 「Q4 マーケティングキャンペーン」）
4. **プロジェクトナレッジ（知識ベース）を追加**します

### プロジェクトナレッジ（知識ベース）

プロジェクトには、Claude が参照すべきファイルやドキュメントを追加できます。

- **対応ファイル形式**: PDF、Word、Excel、テキストファイル、画像、コードファイルなど
- **コンテキストウィンドウ**: 各プロジェクトは最大 200K トークン（約500ページ相当）のコンテキストを保持できます
- **追加方法**: ファイルをドラッグ＆ドロップまたはアップロードボタンから追加します

追加できるコンテンツの例:
- スタイルガイドやブランドガイドライン
- コードベースやAPI仕様書
- インタビュー記録や議事録
- 研究データや分析レポート
- 社内ポリシーや手順書

### カスタム指示の設定

プロジェクトごとに「カスタム指示」（システムプロンプト）を設定できます。これにより、プロジェクト内のすべての会話で Claude の振る舞いを統一できます。

設定例:
- 「このプロジェクトでは常にフォーマルなビジネストーンで回答してください」
- 「回答は必ず日本語で行ってください」
- 「あなたはシニアマーケティングストラテジストとして回答してください」
- 「回答の最後に必ず次のアクションアイテムを3つ提案してください」

### チーム共有機能

Team プラン以上では、プロジェクトをチームメンバーと共有できます。

- **プロジェクトの共有**: 作成したプロジェクトをチームメンバーに公開できます
- **会話のスナップショット**: 優れた回答やプロンプトのパターンをチームで共有し、集合知を蓄積できます
- **アクティビティフィード**: プロジェクト内のチームの活動を確認できます

### プライバシーとデータ保護

- プロジェクト内で共有されたデータやチャットは、ユーザーの明示的な同意なしに AI モデルのトレーニングに使用されません
- Team/Enterprise プランでは、管理者がデータの取り扱いポリシーを設定できます

### 実践のポイント

- **プロジェクトはテーマ別に分ける**: 「採用」「プロダクト開発」「カスタマーサポート」など、明確なテーマで分けると管理しやすくなります
- **ナレッジは最新に保つ**: プロジェクトに追加したファイルが古くなったら更新しましょう。古い情報を参照すると誤った回答の原因になります
- **カスタム指示を活用する**: 毎回同じ前提条件を説明する手間を省けます。「あなたは〇〇です」「回答は〇〇形式で」といった指示をカスタム指示に設定しておきましょう
- **チームのベストプラクティスを蓄積する**: うまくいったプロンプトや回答のパターンを共有して、チーム全体のスキルを向上させましょう

## トピック 5: Artifacts（アーティファクト）

### 何を学ぶか

Artifacts は、Claude が生成するコード、ドキュメント、図表、インタラクティブコンテンツなどを、メインの会話とは別の専用パネルに表示・管理する機能です。生成されたコンテンツの編集、ダウンロード、反復的な改善方法を学びます。

### Artifacts とは

通常、Claude の回答はチャットウィンドウ内にテキストとして表示されます。しかし、コードやドキュメント、図表など「独立したコンテンツ」の場合、会話の中に埋め込まれると見にくく、再利用しにくくなります。

Artifacts は、こうしたコンテンツを会話の右側に表示される専用パネルに分離する機能です。これにより、コンテンツの確認、編集、ダウンロードが容易になります。

### Artifact が生成される条件

Claude は以下の条件を満たすコンテンツを自動的に Artifact として生成します。

- **十分な長さ**: 一般的に15行以上のコンテンツ
- **自己完結性**: 会話のコンテキストなしでも単独で機能するもの
- **再利用の可能性**: 後から参照・編集・ダウンロードする価値があるもの

### 対応コンテンツの種類

| 種類 | 説明 | 例 |
|------|------|-----|
| **マークダウン/テキスト文書** | 構造化されたドキュメント | レポート、記事、マニュアル |
| **コードスニペット** | プログラミングコード | Python、JavaScript、SQLなど |
| **HTML ウェブページ** | 単一ページの Web コンテンツ | ランディングページ、ダッシュボード |
| **SVG 画像** | ベクター画像 | ロゴ、アイコン、図解 |
| **図表・フローチャート** | Mermaid 記法による図 | プロセスフロー、組織図 |
| **React コンポーネント** | インタラクティブな UI | ゲーム、データ可視化、ツール |

### Artifacts の基本操作

**表示と確認**
- Artifact は会話の右側パネルに自動表示されます
- コードの場合、「プレビュー」タブで実行結果を確認できます（HTML、React など）

**編集と改善**
- Claude に「この部分を修正して」と指示することで、Artifact を反復的に改善できます
- 変更は Artifact ウィンドウに直接反映されます
- バージョン履歴が保持されるので、過去のバージョンに戻すこともできます

**エクスポート**
- **コピー**: Artifact の内容をクリップボードにコピーできます
- **ダウンロード**: ファイルとしてローカルに保存できます
- **コードの確認**: 「View code」で生成されたコードのソースを確認できます

**複数の Artifact**
- 1つの会話で複数の Artifact を生成・管理できます
- スライダーアイコンで Artifact 間を切り替えられます

### エラー対応

Artifact のプレビューでエラーが発生した場合、「Try fixing with Claude」ボタンをクリックすると、エラーの詳細が自動的に Claude に送信され、修正を試みます。プログラミングの知識がなくても、Claude がエラーを解析して修正してくれます。

### AI 搭載アーティファクト

公開された Artifact は、閲覧者が直接 Claude と対話して Artifact をカスタマイズできます。例えば、ダッシュボードの Artifact を共有した場合、受け取った人は自分の Claude アカウントでそのダッシュボードのデータや表示を変更できます。

### MCP 統合（Pro/Max/Team/Enterprise 向け）

Model Context Protocol（MCP）を通じて、Artifact から外部サービス（Asana、Google Calendar、Slack など）に接続できます。データの読み取りと書き込みの両方が可能です。

### ストレージ

- 公開された Artifact には最大 20MB の永続ストレージが付与されます
- 個人用ストレージと共有ストレージから選択可能です

### 有効化・無効化

Settings > Capabilities の「Artifacts」トグルで有効・無効を切り替えられます。Team/Enterprise プランでは、組織レベルで設定を管理できます。

### 実践のポイント

- **コーディング知識がなくてもOK**: 「インタラクティブな円グラフを作って」のように自然言語で指示するだけで、Claude がコードを生成し、プレビューで確認できます
- **反復的に改善する**: 最初の生成結果を基に「色を変えて」「データを追加して」「レイアウトを調整して」と少しずつ改善していくのが効果的です
- **プレゼン資料や報告書に活用**: 生成したグラフや図表をダウンロードして、プレゼン資料や報告書に直接貼り付けられます
- **プロトタイプの迅速な作成**: Web アプリやダッシュボードのプロトタイプを数分で作成し、チームに共有できます

## トピック 6: Skills（スキル機能）

### 何を学ぶか

Skills を使って、特定のタスクに対する詳細な指示セットを Claude に提供し、一貫した品質で繰り返しタスクを実行させる方法を学びます。

### Skills とは

Skills は、Claude が特定のタスクをより正確かつ一貫して実行するための「指示、スクリプト、リソースをまとめたフォルダ」です。例えば、「会議議事録を作成する」「Excel レポートを生成する」「プレゼンテーション資料を作る」といった繰り返し行うタスクに対して、詳細な手順やフォーマット、品質基準を事前に定義しておけます。

通常のプロンプトとの違いは、Skills がより構造化されており、再利用可能で、複数のリソース（テンプレート、参考資料、スクリプトなど）をパッケージとして含められる点です。

### Skills の4つの特性

1. **互換性（Composable）**: 複数のスキルを組み合わせて使用できます。例えば、「データ分析」スキルと「レポート作成」スキルを組み合わせて、データの分析結果をレポート形式で出力できます
2. **移植性（Portable）**: Claude アプリ、Claude Code、API のすべてで同じスキルを使用できます
3. **効率性（Efficient）**: 必要な情報だけを読み込むため、コンテキストウィンドウを効率的に使います
4. **強力性（Powerful）**: 単なるテキスト指示だけでなく、実行可能なスクリプトやコードを含めることができます

### 利用可能なプラン

Skills は Pro、Max、Team、Enterprise プランで利用可能です。

### プリセットスキルの活用

Claude には、よく使われるタスクのためのプリセットスキルが用意されています。

- **Excel / スプレッドシート作成**: データの整理や集計表の作成
- **プレゼンテーション作成**: PowerPoint 形式のスライド作成
- **ドキュメント作成**: 報告書やマニュアルの作成
- **データ分析**: CSV データの分析と可視化

### カスタムスキルの作成方法

独自のスキルを作成して、組織固有のワークフローに最適化できます。

**対話的な作成（推奨）**

1. Claude に「このワークフローのスキルを作成して」と依頼します
2. Claude がワークフローについて質問します（入力形式、出力形式、品質基準など）
3. 対話を通じて要件を明確にし、Claude がスキルの構造を自動生成します
4. 生成されたスキルをテストし、必要に応じて調整します

**スキルの構成要素**

- **指示ファイル**: タスクの手順、ルール、品質基準を記述
- **テンプレート**: 出力のフォーマットやレイアウトを定義
- **リソース**: 参考資料、サンプルデータ、ブランドガイドラインなど
- **スクリプト**（オプション）: 自動化されたデータ処理やファイル変換のためのコード

### 実践のポイント

- **繰り返しタスクから始める**: 毎週作成するレポート、定型メールの作成など、繰り返し行うタスクからスキル化を始めましょう
- **組織の標準を反映させる**: ブランドガイドライン、文書フォーマット、品質基準をスキルに組み込むことで、組織全体で一貫した出力を得られます
- **チームで共有する**: 作成したスキルをチームメンバーと共有することで、チーム全体の生産性を向上させられます
- **段階的に改善する**: 最初から完璧なスキルを目指すのではなく、使いながら改善していくアプローチが効果的です

## トピック 7: Enterprise search（エンタープライズ検索）

### 何を学ぶか

組織内のナレッジベースやデータソースを Claude が検索・参照するエンタープライズ検索機能について学びます。社内情報を活用した、より正確で文脈に即した回答を得る方法を理解します。

### エンタープライズ検索とは

エンタープライズ検索は、Claude が組織内部のドキュメント、ナレッジベース、データベースなどを検索し、それらの情報に基づいて回答を生成する機能です。通常の Claude は一般的な知識に基づいて回答しますが、エンタープライズ検索を使うと、「自社の」ポリシー、手順、データに基づいた回答が得られます。

### コネクタ（Connectors）

エンタープライズ検索は「コネクタ」を通じて、さまざまなデータソースに接続します。

**対応データソース例**:
- Google Drive / Google Docs
- Confluence
- Notion
- SharePoint
- Salesforce
- GitHub
- その他の企業向けツール

### 利用要件

- **対象プラン**: Enterprise プラン（一部機能は Team プランでも利用可能）
- **管理者による設定**: データソースとの接続は組織の管理者が設定します
- **アクセス権限**: ユーザーのアクセス権限に基づいて検索結果がフィルタリングされます。各ユーザーは自分がアクセス権を持つ情報のみを検索・閲覧できます

### 使い方

エンタープライズ検索が設定されている場合、通常の会話の中で自然に社内情報を参照できます。

例:
- 「当社の休暇ポリシーについて教えて」
- 「前四半期の売上レポートの要点をまとめて」
- 「新入社員オンボーディングの手順を教えて」

Claude は自動的に接続されたデータソースを検索し、関連する社内ドキュメントに基づいて回答します。

### セキュリティとプライバシー

- **権限ベースのアクセス**: ユーザーがアクセス権を持つドキュメントのみ検索対象になります
- **データの分離**: 異なる組織のデータは完全に分離されています
- **トレーニング不使用**: エンタープライズ検索で参照されたデータは AI モデルのトレーニングに使用されません
- **監査ログ**: 管理者はアクセスログを確認できます

### 実践のポイント

- **自然言語で質問する**: 検索クエリを考える必要はありません。普通に質問するだけで Claude が適切な社内ドキュメントを見つけます
- **情報の鮮度を確認する**: Claude が参照した情報がいつのものかを確認しましょう。「この情報のソースと日付は？」と聞けば確認できます
- **フィードバックする**: 回答が不正確な場合は、正しい情報源やドキュメントを教えてあげることで、次回以降の精度が向上します

## トピック 8: Research Mode（リサーチモード）

### 何を学ぶか

リサーチモードは、Claude がエージェンティック（自律的）に複数のソースを検索・分析し、包括的な調査レポートを作成する機能です。Web 検索と内部データの両方を活用した高度な調査方法を学びます。

### リサーチモードとは

通常の Claude の会話では、Claude は自身の学習済み知識に基づいて回答します。リサーチモードを有効にすると、Claude は自律的にウェブ検索を実行し、複数のソースから情報を収集・統合して、より包括的で最新の情報を含む回答を提供します。

単純な1回の検索ではなく、Claude が自ら「次に何を調べるべきか」を判断しながら、複数の検索を連鎖的に実行するのが特徴です。調査は数分間にわたって行われることもあり、最大45分かけて徹底的に調査するケースもあります。

### 主な特徴

**エージェンティックな検索**
- Claude が自律的に検索クエリを生成し、実行します
- 最初の検索結果に基づいて、さらに深掘りする質問を自動的に生成します
- 異なる角度から情報を探索し、多面的な視点を提供します

**包括的なレポート生成**
- 調査結果を構造化されたレポートとして出力します
- 各情報には検証可能な引用・出典が付きます
- 矛盾する情報がある場合は、その旨を明示します

**Google Workspace 連携**
- Gmail、Google Calendar、Google ドキュメントと統合できます
- Web 上の情報と社内の情報を組み合わせた調査が可能です
- ファイルの手動アップロードなしで社内情報を参照できます

### 利用可能なプラン

- **Web 検索**: すべてのプランで利用可能（グローバル）
- **リサーチモード（高度な調査）**: Max、Team、Enterprise プラン
- **Google Workspace 連携**: Team、Enterprise プランで設定可能

### 使い方

1. チャット画面でメッセージ入力欄の近くにある「Research」ボタンまたはトグルをオンにします
2. 調査したい内容を自然言語で入力します（例: 「2025年の日本の再生可能エネルギー市場の動向を調査して」）
3. Claude が自動的に複数の検索を実行し始めます
4. 検索の進捗がリアルタイムで表示されます
5. 調査が完了すると、引用付きの包括的なレポートが生成されます

### 活用例

| ユースケース | 具体例 |
|-------------|--------|
| **市場調査** | 競合企業の最新動向、業界トレンドの分析 |
| **顧客会議の準備** | 顧客企業の最新ニュース、財務情報、業界ポジションの調査 |
| **技術調査** | 新しい技術やフレームワークの比較調査 |
| **政策・規制の調査** | 最新の法規制や業界基準の確認 |
| **製品ローンチの準備** | 市場環境、競合製品、顧客ニーズの総合的な調査 |

### 実践のポイント

- **具体的な質問をする**: 「AI について教えて」よりも「2025年の日本企業における生成AI導入状況と課題を調査して」の方が、はるかに有用なレポートが得られます
- **調査の範囲を指定する**: 「過去6ヶ月の情報に限定して」「日本市場に焦点を当てて」など、範囲を限定すると的確な結果が得られます
- **引用を確認する**: リサーチモードの結果には引用が付いています。重要な意思決定に使う場合は、元ソースを確認しましょう
- **反復的に深掘りする**: 最初のレポートを基に「この点についてもっと詳しく」と追加の調査を依頼できます
