# クラウド連携コース

[Claude with Amazon Bedrock](https://anthropic.skilljar.com/claude-in-amazon-bedrock) | [Claude with Google Cloud's Vertex AI](https://anthropic.skilljar.com/claude-with-google-vertex)

## 概要

Anthropic Academy では、主要クラウドプラットフォーム経由で Claude を利用するための2つのコースが提供されています。企業のセキュリティ要件やインフラ要件に合わせて、AWS または Google Cloud 経由で Claude を利用する方法を学べます。

直接 Anthropic API を使う方法に加えて、既存のクラウドインフラに Claude を統合する選択肢があることは、エンタープライズ環境において非常に重要です。

### 共通の学習目標

両コースとも以下の実践的なスキルを習得します:

- マルチターン会話とシステムプロンプト設定による Claude API の実装
- 構造化手法によるプロンプトの構築と評価
- JSON Schema によるカスタムツール定義とファンクションコーリング
- テキストチャンキング、エンベディング、BM25検索を用いた RAG パイプラインの構築
- 拡張思考（Extended Thinking）、ビジョン機能、プロンプトキャッシュなどの高度な機能の設定
- Claude Code による自動デバッグとタスク自動化
- Model Context Protocol（MCP）によるツールとリソースの定義
- ストリーミング、温度制御、構造化データ抽出によるインファレンスの最適化
- モデルベースおよびコードベースの評価フレームワークの構築

### 対象者

- バックエンド開発者（AI 駆動のアプリケーション構築）
- ML エンジニア（本番 RAG システムの実装）
- DevOps エンジニア（Claude モデルのデプロイ）
- フルスタック開発者（ツール利用とエージェントアプリケーション）
- テクニカルアーキテクト（スケーラブルな AI システム設計）
- 自動化エンジニア（自律エージェントの構築）

## コース1: Claude with Amazon Bedrock

[受講する](https://anthropic.skilljar.com/claude-in-amazon-bedrock)

### Amazon Bedrock とは

Amazon Bedrock は、AWS が提供するフルマネージドのファウンデーションモデルサービスです。Claude を含む複数の AI モデルに、単一の API でアクセスできます。

### 前提条件

- Python プログラミングの習熟
- AWS サービスと Amazon Bedrock の基礎知識
- AWS アカウント

### セットアップと認証

#### 1. AWS CLI のインストールと設定

```bash
# AWS CLI バージョン 2.13.23 以降をインストール
# https://docs.aws.amazon.com/cli/latest/userguide/cli-chap-welcome.html

# 認証情報の設定
aws configure

# 認証情報の確認
aws sts get-caller-identity
```

#### 2. Bedrock モデルアクセスの有効化

[AWS Console > Bedrock > Model Access](https://console.aws.amazon.com/bedrock/home?region=us-west-2#/modelaccess) から Anthropic モデルへのアクセスをリクエストします。

#### 3. SDK のインストール

Anthropic のクライアント SDK は Bedrock をサポートしています。AWS SDK（boto3）を直接使用することもできます。

**Python:**
```bash
pip install -U "anthropic[bedrock]"
```

**TypeScript:**
```bash
npm install @anthropic-ai/bedrock-sdk
```

**Go:**
```bash
go get github.com/anthropics/anthropic-sdk-go/bedrock
```

**Java (Gradle):**
```groovy
implementation("com.anthropic:anthropic-java-bedrock:2.18.0")
```

**Boto3 (Python):**
```bash
pip install boto3>=1.28.59
```

#### 4. IAM 認証

Bedrock では複数の認証方式をサポートします:

- **AWS 認証情報プロバイダーチェーン**: `~/.aws/credentials`、環境変数（`AWS_SECRET_ACCESS_KEY`、`AWS_ACCESS_KEY_ID`）
- **一時認証情報**: `aws_session_token` を使用
- **ベアラートークン認証**: 企業環境で AWS 認証情報の管理なしにアクセス可能（C#、Go、Java SDK でサポート）。`AWS_BEARER_TOKEN_BEDROCK` 環境変数で設定

### API 呼び出しの基本

#### Anthropic SDK を使用（推奨）

```python
from anthropic import AnthropicBedrock

client = AnthropicBedrock(
    aws_access_key="<access key>",
    aws_secret_key="<secret key>",
    aws_region="us-west-2",
)

message = client.messages.create(
    model="global.anthropic.claude-opus-4-6-v1",
    max_tokens=256,
    messages=[{"role": "user", "content": "Hello, world"}],
)
print(message.content)
```

#### Boto3 を使用

```python
import boto3
import json

bedrock = boto3.client(service_name="bedrock-runtime")
body = json.dumps({
    "max_tokens": 256,
    "messages": [{"role": "user", "content": "Hello, world"}],
    "anthropic_version": "bedrock-2023-05-31",
})

response = bedrock.invoke_model(
    body=body,
    modelId="global.anthropic.claude-opus-4-6-v1"
)
response_body = json.loads(response.get("body").read())
print(response_body.get("content"))
```

#### TypeScript

```typescript
import AnthropicBedrock from "@anthropic-ai/bedrock-sdk";

const client = new AnthropicBedrock({
  awsRegion: "us-west-2"
});

const message = await client.messages.create({
  model: "global.anthropic.claude-opus-4-6-v1",
  max_tokens: 256,
  messages: [{ role: "user", content: "Hello, world" }]
});
```

### グローバル vs リージョナルエンドポイント

Claude Sonnet 4.5 以降のモデルで、2種類のエンドポイントが利用可能です:

**グローバルエンドポイント（推奨）:**
- 動的ルーティングで最大の可用性を提供
- 追加料金なし
- データレジデンシーの柔軟性がある場合に最適
- モデル ID に `global.` プレフィックスを使用

```python
# グローバルエンドポイント
model="global.anthropic.claude-opus-4-6-v1"
```

**リージョナルエンドポイント（CRIS）:**
- 特定の地理的リージョンを通じてトラフィックをルーティング
- データレジデンシーとコンプライアンス要件に必須
- US、EU、日本、オーストラリアで利用可能
- 10% の価格プレミアム

```python
# リージョナルエンドポイント（global. プレフィックスなし）
model="anthropic.claude-opus-4-6-v1"

# または US リージョナル
model="us.anthropic.claude-opus-4-6-v1"
```

### Converse API

Bedrock の Converse API は、複数のモデル間で統一されたインターフェースを提供します。Claude 固有の API に加えて、Converse API を通じてモデル間でのコード共有が容易になります。

### エンタープライズ機能

#### VPC エンドポイントによるプライベートアクセス

VPC エンドポイントを使用して、インターネットを経由せずに Bedrock にアクセスできます。プライベートサブネット内のリソースから Claude を安全に呼び出せます。

#### CloudWatch によるモニタリングとログ

Bedrock は[呼び出しログサービス](https://docs.aws.amazon.com/bedrock/latest/userguide/model-invocation-logging.html)を提供しています。プロンプトと完了をログに記録できます。

**推奨**: 少なくとも30日間のローリングベースでアクティビティをログ記録し、使用状況の把握と不正使用の調査に備える。

> ログサービスを有効にしても、AWS や Anthropic がコンテンツにアクセスすることはありません。

#### コンプライアンスとセキュリティ

- AWS のコンプライアンスフレームワーク（SOC 2、HIPAA 等）に準拠
- IAM によるきめ細かなアクセス制御
- データ暗号化（転送中・保管中）

#### コスト管理

- 従量課金でスケーラブル
- AWS Cost Explorer によるコスト分析
- 使用量のモニタリングとアラート設定

#### PDF サポート

Bedrock では Converse API と InvokeModel API の両方で PDF サポートが利用可能です。

- **Converse API**: ビジュアル PDF 分析（チャート、画像、レイアウト）にはサイテーション有効化が必要。サイテーションなしではテキスト抽出のみ
- **InvokeModel API**: 強制サイテーションなしでフル制御可能

#### コンテキストウィンドウ

Claude Opus 4.6、Sonnet 4.6、Sonnet 4.5、Sonnet 4 は Amazon Bedrock で **1M トークンのコンテキストウィンドウ**を利用可能（Sonnet 4.5 と 4 はベータ。`context-1m-2025-08-07` ベータヘッダーが必要）。

リクエストペイロードの上限は **20 MB** です。

### Bedrock を選ぶ理由

- 既存の AWS インフラとの統合が容易
- AWS のセキュリティ・コンプライアンスの恩恵を受けられる
- 他の AWS サービス（S3、Lambda、SageMaker 等）との連携
- 従量課金でスケーラブル
- Converse API によるモデル間の統一インターフェース

## コース2: Claude with Google Cloud's Vertex AI

[受講する](https://anthropic.skilljar.com/claude-with-google-vertex)

### Vertex AI とは

Vertex AI は、Google Cloud が提供する統合 ML プラットフォームです。Claude を含む外部モデルを、Google Cloud の環境内で利用できます。

### 前提条件

- Python プログラミングの習熟
- Google Cloud Platform の経験
- JSON データ構造の理解
- Google Cloud アカウント

### Vertex AI 固有の学習内容

Bedrock コースの共通学習目標に加えて、以下の内容が含まれます:

- PDF 処理、サイテーション生成
- Computer Use による UI 自動化
- 並列化、チェーン、ルーティングパターンによるエージェントベースのワークフロー設計

### セットアップと認証

#### 1. SDK のインストール

**Python:**
```bash
pip install -U google-cloud-aiplatform "anthropic[vertex]"
```

**TypeScript:**
```bash
npm install @anthropic-ai/vertex-sdk
```

**Go:**
```bash
go get github.com/anthropics/anthropic-sdk-go
```

**Java (Gradle):**
```groovy
implementation("com.anthropic:anthropic-java-vertex:2.18.0")
```

#### 2. 認証

Google Cloud のデフォルト認証情報を使用します:

```bash
gcloud auth application-default login
```

サービスアカウントを使用する場合は、適切な IAM ロール（Vertex AI ユーザー等）を付与し、認証キーを設定します。

#### 3. API の違い

Vertex AI の API は Messages API とほぼ同一ですが、2つの重要な違いがあります:

- **model はリクエストボディに含まない** — Google Cloud エンドポイント URL で指定
- **anthropic_version はリクエストボディで指定** — 値は `vertex-2023-10-16`

### API 呼び出しの基本

#### Python SDK

```python
from anthropic import AnthropicVertex

project_id = "MY_PROJECT_ID"
region = "global"

client = AnthropicVertex(project_id=project_id, region=region)

message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=100,
    messages=[
        {"role": "user", "content": "Hey Claude!"}
    ],
)
print(message)
```

#### TypeScript SDK

```typescript
import { AnthropicVertex } from "@anthropic-ai/vertex-sdk";

const client = new AnthropicVertex({
  projectId: "MY_PROJECT_ID",
  region: "global"
});

const result = await client.messages.create({
  model: "claude-opus-4-6",
  max_tokens: 100,
  messages: [{ role: "user", content: "Hey Claude!" }]
});
```

#### cURL（直接 API 呼び出し）

```bash
MODEL_ID=claude-opus-4-6
LOCATION=global
PROJECT_ID=MY_PROJECT_ID

curl \
  -X POST \
  -H "Authorization: Bearer $(gcloud auth print-access-token)" \
  -H "Content-Type: application/json" \
  https://$LOCATION-aiplatform.googleapis.com/v1/projects/${PROJECT_ID}/locations/${LOCATION}/publishers/anthropic/models/${MODEL_ID}:streamRawPredict \
  -d '{
    "anthropic_version": "vertex-2023-10-16",
    "messages": [{"role": "user", "content": "Hey Claude!"}],
    "max_tokens": 100
  }'
```

## どちらを選ぶべきか

どちらのプラットフォームを選択しても、Claude の基本的な機能（Messages API のパラメータ、ツール利用、ストリーミング、拡張思考等）は同じです。選択の判断基準:

1. **既存のクラウドインフラ**: AWS 中心なら Bedrock、GCP 中心なら Vertex AI
2. **データレジデンシー要件**: 必要なリージョンでの対応状況を確認
3. **他サービスとの連携**: 既存のデータパイプライン、ML ワークフローとの統合しやすさ
4. **認証基盤**: 既存の IAM/サービスアカウント基盤の活用
5. **プロビジョンドスループット**: 安定した処理量が必要なら Vertex AI のリージョナルエンドポイント

### ベストプラクティス

- **グローバルエンドポイントを優先使用** — 特別なデータレジデンシー要件がない限り、グローバルエンドポイントを推奨（追加料金なし、高可用性）
- **アクティビティログを有効化** — 最低30日間のログを維持
- **最小権限の原則** — IAM ロール/サービスアカウントに必要最小限の権限を付与
- **バージョン固定** — 本番環境ではスナップショット日付付きのモデル ID を使用して安定性を確保
- **エラーハンドリング** — リトライロジックとレート制限への対応を実装
- **コスト監視** — 予算アラートを設定し、使用量を定期的に確認
